# DankDash — Short Agent Instructions

Goal: Maintain and build a modern, mobile-first platform with:
- Public site (shop, delivery, partner onboarding, contact).
- Private Odoo-like backend (CRM, sales, products, accounting, marketing, delivery, grow monitoring).
- Partner API for Delivery-as-a-Service (orders, shipping, tracking, labels).
- Always responsive, clean, centered design.

Key Requirements:
1. **UI** – Minimalist, rounded corners, consistent spacing, responsive.
2. **Public** – Real-time chatbot sales agent, cart/checkout, local + nationwide delivery.
3. **Backend** – Upload/manage products (photos, videos, docs), track customers/orders, assign deliveries.
4. **Partner API** – `POST /orders`, `GET /orders`, `POST /rates`, `POST /shipments`, HMAC-secured webhooks, 60 req/min.
5. **Automation** – Auto post to social media, marketing campaigns, grow room monitoring, sales/chat automation.
6. **Agents** – Builder Agent (read instructions), Marketing Agent, Sales Agent, Delivery Agent, Research Agent, Grow Monitor Agent.
7. **Deployment** – GitHub repo, branch protections, Vercel auto-deploy.

Never break style consistency. Default to centered, modern, clean UI unless instructed otherwise.
